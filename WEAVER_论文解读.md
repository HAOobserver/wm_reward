## 一、研究动机:世界模型的"三难困境"

**世界模型(World Model, WM)** 就是一个学出来的模拟器:输入当前观测和一串候选动作,预测未来会发生什么。它对机器人有三大用途:

1. **策略评估**:不跑真机就能估计策略的成功率(真机测试昂贵且不安全);
2. **策略改进**:在"想象"中生成合成数据,用来微调策略;
3. **测试时规划**:执行前先在脑内推演多个候选动作,挑最好的执行。

要同时支撑这三件事,WM 必须同时满足三个核心要求,这正是标题的含义:

| 要求 | 标题对应 | 含义 |
|---|---|---|
| Fidelity(保真度) | Better | 预测物理上准确,与真实结果相关 |
| Efficiency(效率) | Faster | 生成速度快,能满足实时规划 |
| Consistency(一致性) | Longer | 长时程滚动预测不漂移、不崩坏 |

**现有方法没有一个能三者兼得**:

- **视频生成类 WM**:画面保真,但推理太慢,没法做规划;
- **JEPA 类**(纯潜空间预测):快,但潜状态解码不回图像。而评估任意视觉运动策略(VLA)必须把预测图像喂回策略生成下一步动作,所以走不通;
- **Dreamer-v4**:思路接近,但视觉 encoder 从零训练,对分布外(OOD)输入鲁棒性差;
- **Ctrl-World**(此前操作领域最强基线,1.5B 参数、从 Stable Video Diffusion 初始化):多视角、保真度不错,但生成 10 秒片段要 40 多秒。

操作(manipulation)场景额外难点:多相机视角、需从历史推断被遮挡物体(夹爪经常挡住目标)、要求"世界状态正确"而不只是"画面好看"。

## 二、方法:WEAVER 的设计

WEAVER(World Estimation Across Views for Embodied Reasoning)不发明全新组件,而是**把多个社区的关键技术缝合成一个整体**:

- 从**视频生成社区**:flow matching + diffusion forcing(长时程 + 快推理)、预训练 VAE encoder(OOD 鲁棒);
- 从**潜空间世界模型**(Dreamer-v4 一脉):潜空间 reward/critic 头,打分不需要外部 VLM 裁判;
- 从 **JEPA**:训练目标是预测未来潜变量,而非重建像素;
- 从 **Ctrl-World**:多视角生成 + 记忆架构。

### 2.1 问题设定与记号

- 观测 `o_t = (I_t, q_t)`:n 个 RGB 视角(外部相机 + 腕部相机)+ 本体感知状态 `q`(8 维:7 关节角 + 夹爪);
- 策略 `pi_theta` 每次输出 **h 步动作块** `a[t:t+h]`;
- 观测经预训练 encoder 编码成潜状态:`z_t ~ E_psi(o_t)`。

### 2.2 架构:四个关键设计决策

**(1)多视角预测。** 同时预测外部相机和腕部相机,两视角互补以对抗遮挡。每个视角用**预训练 Stable Diffusion 3 VAE encoder** 编成 H×W 个 patch token,本体感知状态投影成一个 token,拼接得 `z_t`。预训练 encoder 带来 OOD 鲁棒性;配套 decoder 保证潜状态能解码回图像喂给任意 VLA 策略(与 JEPA 的本质区别)。

**(2)显式预测本体感知状态。** 与 Ctrl-World 不同,还预测未来关节位置和夹爪宽度,对接触密集的可变形物体操作至关重要。

**(3)稀疏记忆 + 短期历史。**

```
z_mem[t]  = ( ..., z[t-2k], z[t-k] )     # 每隔 k 帧取一帧,长期上下文
z_hist[t] = ( z[t-1], z[t] )             # 最近两帧,动作的即时后果
```

记忆解决"物体离开视野再回来还认得",历史捕捉短期动力学。

**(4)潜动力学模型。**

```
z_hat[t+1 : t+h+1]  ~  f_phi( · | z_mem[t], z_hist[t], a[t:t+h] )
```

32 层、1536 维、16 头的 2D transformer,每层 = 空间注意力(同帧 patch 间)+ 因果时间注意力(跨时间步),配 RMSNorm、RoPE、QKNorm、SwiGLU。整个模型 **928M 参数**(小于 Ctrl-World 的 1.5B)。

### 2.3 训练目标:flow matching + diffusion forcing

```
x1 = z[t+1 : t+h+1]                    # 真实的未来 h 帧潜变量
x0 ~ N(0, I)                           # 同维度高斯噪声
x_tau = tau * x1 + (1 - tau) * x0      # 插值, tau ∈ [0, 1)

L_WM = E[ || (x1 - x0) - f_phi(z_hist, z_mem, a, x_tau, tau) ||^2 ]
```

模型学从噪声指向数据的"速度场" `x1 - x0`,损失是 MSE。两个附加技巧:

- **Diffusion Forcing**:训练时每个未来时间步独立采样噪声水平 tau,让模型习惯"部分干净、部分带噪"的条件——自回归长时程滚动不崩的关键;
- **SPRINT blocks**:以 0.5 概率丢弃 patch token,降低计算量。

### 2.4 推理加速(Faster 的三板斧)

1. **KV 缓存**:去噪迭代中记忆/历史 token 不变(噪声水平恒为 1),key/value 只算一次,省约 30% 时间;
2. **余弦噪声调度**:

```
Linear :  k = i / K
Sigmoid:  k = sigmoid( alpha * (i/K - 0.5) )
Power  :  k = (i / K)^0.5
Cosine :  k = 1 - cos( i * pi / (2K) )
```

cosine/power 把更多去噪预算花在低噪声区(精细细节阶段),消融显示优于 Ctrl-World 用的 linear;

3. **Rectified Flow 后训练(WEAVER-ReFlow)**:冻结微调好的模型当老师,完整去噪生成高质量潜轨迹 `x1_hat` 当回归目标蒸馏学生:

```
L_ReFlow = E[ || (x1_hat - x0) - f_phi_student(z_hist, z_mem, a, x_tau, tau) ||^2 ]
```

学生学到"拉直"的生成路径,NFE=4 就接近原来 NFE=16 的质量。

### 2.5 潜空间奖励模型与 Critic

把现成奖励模型 Robometer 的打分蒸馏进直接作用于潜状态的轻量 reward head(AdaPool + CLIP 指令 embedding + MLP,MSE 训练);再训一个 critic 估计想象视野之外的价值(λ-return 目标,gamma=0.995、lambda=0.95)。推理时各只需约 0.0006 秒。**详见第二部分 Q1/Q2/Q4。**

### 2.6 三个下游应用的做法

- **策略评估**:真实动作序列在 WM 中开环重放,自回归想象整条轨迹(长任务需迭代调用 40+ 次),对比想象与真实的成功率;
- **策略改进**:从同一初始状态采 B 条想象 rollout(共 H = K*h 步),算 H 步优势:

```
A_hat[b] =  sum_{j=1..H} gamma^(j-1) * R(z_hat[b][t+j], instr)
          + gamma^H * V(z_hat[b][t+H], instr)
          - V(z[t], instr)
```

取最优 `b* = argmax_b A_hat[b]`,**仅当 A_hat[b*] > 0.1** 才蒸馏进策略(advantage filtering);

- **测试时规划**:单块 best-of-N——策略采 B=4 个候选动作块,WM 想象结果,优势打分,执行最高分的块。

## 三、实验与结果

### 3.1 实验设置

- **基础策略**:π0.5(DROID 上训练的 SOTA 视觉-语言-动作模型);
- **硬件**:Franka Emika Panda,DROID 配置(实际用右相机 + 腕部相机);
- **五个任务**(基础策略成功率至少 20%,覆盖不同操作类型):Stack Bowls(叠碗,刚体)、PnP Bag(薯片袋放盘,可变形)、PnP Marker(笔重定向插杯,精细抓取)、PnP Towel(毛巾入篮,可变形)、Pour Beans(倒咖啡豆,动态/颗粒);
- **训练**:DROID 预训练 1M 步(4×H100,10 天);任务数据(每任务 50 条,共 250 条)微调 16k 步得 WEAVER-FT;
- **工程细节**:想象频率 5Hz;π0.5 输出关节速度,专门训了 3 层 MLP 的 **action adapter** 转成关节位置增量(夹爪损失加权 5 倍)。

### 3.2 世界模型质量:Pareto 支配 Ctrl-World

DROID 验证集、10 秒 rollout、NFE=16:外部视角 **FID 10.20 vs Ctrl-World 26.09**,推理时间 **4.78s vs 14.65s**。

- 降 NFE 时 Ctrl-World 质量崩得快,WEAVER 几乎不掉:WEAVER 用 NFE=8(2.53s)已好于 Ctrl-World 用 NFE=50(42.33s),等效约 **16 倍加速**;
- 长时程逐段 FID 始终更低(diffusion forcing + 记忆/历史的功劳);
- 微调后 WEAVER-FT 仍全面领先公平微调过的 Ctrl-World-FT;ReFlow 版 NFE=4 接近 FT 版 NFE=16。
- 共同短板:腕部视角误差最大(视角连续变化)。

### 3.3 策略评估:与真实成功率相关性 ρ = 0.87

| 模型 | RMSE | Spearman | Pearson | MMRV |
|---|---|---|---|---|
| Ctrl-World | 0.410 | 0.523 | 0.552 | 0.215 |
| WEAVER(零样本) | 0.359 | 0.594 | 0.563 | 0.155 |
| WEAVER-FT | 0.188 | 0.870 | 0.863 | 0.035 |

预训练模型普遍低估策略性能(尤其 Pour Beans,颗粒动力学在 DROID 中代表性不足);微调后大幅改善,可用于替代大量真机测试做策略排序。

### 3.4 策略改进:合成数据几乎等价于真实数据

每任务 20 次真机试验的平均成功率:

- 基础 π0.5:**0.44**
- 真实数据微调(1000 段/任务):**0.71**
- **纯合成数据**微调(WM 想象 + 优势过滤):**0.67**(只差 4%,零真机交互)
- 真实 + 合成混合(2000 段/任务):**0.82**(摘要里的 +38%)

**Scaling 实验**(Pour Beans):合成数据 1k → 2k → 5k 段,成功率 0.55 → 0.60 → **0.70,反超真实数据微调的 0.60**。合成数据可以"无限印",这是世界模型路线最核心的卖点。

### 3.5 测试时规划:+14% 成功率,单次决策约 1.45 秒

B=4、h=12 的 best-of-N:平均成功率 **0.44 → 0.58**(单任务最高 +20%)。A6000 上延迟分解:策略采样 0.20s + 动力学想象 1.25s(瓶颈)+ reward/critic 各约 0.0006s。同配置 Ctrl-World 光动力学就要 29.4s,**快约 20 倍**。规划提升小于微调,因为延迟限制下只能对单个动作块贪心选择。

## 四、局限(论文自述)

1. **部分可观测**:纯视觉看不到接触力、抓取稳定性,可能需要触觉/力觉/深度;
2. **可变形与颗粒动力学难**:小误差随时间复合放大,可能需要物理先验;
3. **规划视野受限**:延迟只允许单动作块选择,无法多阶段前瞻;
4. **数据覆盖单一**:预训练只用 DROID,绑定单一机器人形态;
5. **奖励监督有噪声**:Robometer 分不清视觉相似但语义不同的结果,靠 0.1 优势阈值兜底。

## 五、总结评价

论文价值不在单点创新,而在**系统性证明"保真、一致、高效"可以兼得,且兼得后世界模型的三大承诺第一次在同一真机系统上全部兑现**。三条主线:

1. **预训练 VAE 潜空间 + flow matching + diffusion forcing**,解决"又好又长";
2. **KV cache + cosine 噪声调度 + ReFlow 蒸馏**,解决"又快"(最高 20 倍加速);
3. **潜空间 reward/critic 替代 VLM 裁判 + 优势过滤**,让想象数据安全反哺策略——纯合成数据只比真实数据差 4%,且能靠加量反超,可能是对领域影响最深远的结果。

## Q1: 潜空间奖励模型与 Critic 具体是怎么回事?

> 原问题:"具体讲一下这部分'2.5 潜空间奖励模型与 Critic(甩掉 VLM 裁判)'"

**核心思想:把"给想象轨迹打分"从"解码图像 + 请外部 VLM 当裁判"变成"在潜空间跑两个小 MLP 头",打分成本从秒级降到亚毫秒级。**

旧管线 vs 新管线:

```
旧: 潜状态 --(VAE 解码)--> 图像帧 --(发给 VLM 裁判)--> 好/坏      秒级
新: 潜状态 --(reward head R)--> 每步奖励                         0.0006 秒
    潜状态 --(critic V)------> 视野外价值                        0.0005 秒
```

**Reward head 结构**:

```
z_hat 的所有 patch token --> AdaPool 聚合成向量
                                |
                                拼接 CLIP(instr) 指令 embedding
                                |
                                2 层 MLP --> 标量奖励 r_hat
```

- AdaPool:自适应池化,变长 token 集合压成固定向量,抗噪;
- 拼 CLIP embedding 使奖励**任务条件化**:同一画面在不同指令下进度分不同;
- 可行性前提:WEAVER 潜状态本就训练成"可解码回完整观测",任务进度所需信息已在 latent 里。

**Critic 为什么必要——视野截断问题**:想象只有 h=12 步(约 2.4 秒),任务长 40 秒,只看视野内奖励会短视。critic 估计"从此状态出发未来还能拿多少回报",给视野末端兜底。训练目标是自举 λ-return:

```
v_lam[t]   = R(z[t], instr) + gamma * ( (1-lambda) * V(z[t+1], instr) + lambda * v_lam[t+1] )
v_lam[t+k] = V(z[t+k], instr)          # 边界:末端用 critic 自己估值

L_critic = || V(z[t], instr) - v_lam[t] ||^2
```

lambda=0.95 混合"多步真实奖励"(低偏差)与"critic 一步估值"(低方差);gamma=0.995 的有效视野约 1/(1-gamma)=200 步,5Hz 下约 40 秒,恰好覆盖任务时长。

**组合成优势打分器**(策略改进与规划共用):

```
A_hat[b] =  sum_{j=1..H} gamma^(j-1) * R(z_hat[b][t+j], instr)   # 视野内:逐步奖励
          + gamma^H * V(z_hat[b][t+H], instr)                    # 视野外:critic 兜底
          - V(z[t], instr)                                       # 基线:比"按现状预期"好多少
```

**效果**(附录 Table 8,预测奖励 vs 真值):微调后 RMSE 0.188、Spearman 0.870、MMRV 0.035——用它排序几乎等价于用真实结果排序。

**代价**:天花板由 Robometer 决定,其噪声(如分不清"笔插进杯子"和"放在杯边")会蒸馏进来,靠 0.1 优势阈值兜底(宁可少选、不选错)。

## Q2: "把 Robometer 蒸馏进潜空间"这一步怎么训练?

> 原问题:"'2. 奖励标签从哪来:把 Robometer 蒸馏进潜空间'这一步怎么训练呢"

本质是**两阶段离线蒸馏**:先用老师模型把数据集标一遍,再在潜空间做监督回归。世界模型本体和 VAE encoder 都不动,只训两个小头。

**第一阶段:Robometer 离线标注**

```
对每条轨迹(DROID 预训练集 + 自采任务数据):
  1. 视频抽帧,降采样到 1 fps,只取右相机视角
  2. 帧序列 + 语言指令一次前向发给 Robometer 服务器
  3. 返回逐帧进度分 r ∈ [0, 1](任务完成几成)
  4. 线性插值把奖励序列拉回原帧率
  5. 取"进度分"(比成功概率更贴合实际任务结果)
  6. 整体减 1,得 r ∈ [-1, 0] 作为标签
```

减 1 的直觉:没完成 = 负奖励(每步都在扣分),完成趋近 0,累积回报天然鼓励尽快推进,且价值函数有界。标注只做一次,推理时完全不需要 Robometer。

**第二阶段(a):reward head 监督回归**(输入是 encoder 编码**真实帧**的 latent):

```
vec   = AdaPool(z[t] 的 tokens)
feat  = concat(vec, CLIP_text(instr))
r_hat = MLP_2layer(feat)

L_R = ( r_hat - r_label[t] )^2          # 纯 MSE,没有花样
```

**第二阶段(b):critic 回归**(结构同上,目标由 λ-return 递归算出;注意目标里的奖励用 **R 的输出**而非原始标签,保证推理时组合自洽):

```
# gamma = 0.995, lambda = 0.95, 训练窗口长度 8
r[t+j] = R(z[t+j], instr)                        # 1. 当前 R 打分
v_lam[t+k] = V(z[t+k], instr)                    # 2. 末端边界
for j = k-1 down to 0:                           #    反向递归
    v_lam[t+j] = r[t+j] + gamma * ( (1-lambda) * V(z[t+j+1], instr)
                                    + lambda   * v_lam[t+j+1] )
L_critic = ( V(z[t], instr) - v_lam[t] )^2       # 3. 目标不回传梯度
```

用 λ-return 而非蒙特卡洛回报的原因:训练窗口只有 8 步,覆盖不了 40 秒任务,必须靠末端 V 自举接上"窗口之外的未来"。

**论文未明说但值得注意的点**:

1. **训练/推理分布差**:头在"真实帧编码的 latent"上训练,推理却用在"想象 latent"上;隐含假设是 flow matching 使想象 latent 足够接近真实分布(Table 8 的 0.87 说明实践中成立);
2. 目标计算的梯度处理(target network 等)论文未写明,伪代码按标准做法呈现;
3. 老师的噪声会原样蒸馏进来;
4. 成本极低:两个头只是"池化 + 2 层 MLP",相比 WM 本体 10 天的预训练可忽略。

一句话:**Robometer 当一次性"标注员",reward head 学它的逐帧打分(MSE),critic 在其上用 λ-return 自举学"剩余回报"(MSE)——都是最朴素的监督回归,巧思全在标签构造和输入空间(潜空间 + 指令)的选择上。**

## Q3: Robometer 这个 reward 模型什么水平?

> 原问题:"Robometer 这个 reward 模型什么水平"

**定位:RSS 2026,USC/UT Dallas/MIT/UW/Ai2/NVIDIA 联合出品(作者含 Luke Zettlemoyer、Dieter Fox、Abhishek Gupta 等),是当时通用机器人奖励模型的 SOTA,断层领先,但细粒度判断仍不完美。**(arXiv 2603.02115,项目页 robometer.github.io)

**它是什么**:

- 基于 Qwen3-VL-4B-Instruct,加三个头:逐帧**进度**(0~1)、逐帧**成功概率**、双轨迹**偏好比较**。WEAVER 用的是进度头;
- 训练数据 **RBM-1M**:100 万+ 条轨迹、21 种机器人形态,特意包含大量次优/失败数据(14 万+ 混合质量);
- **双目标训练**:专家轨迹上做逐帧进度回归(锚定刻度)+ 构造偏好对做排序学习(专家 vs 次优、正确任务 vs 错误指令、原视频 vs 倒放伪造的失败)。失败轨迹没法标逐帧进度,但"不如专家"的排序信号免费,因此能吃海量脏数据。

**硬指标**(分布外评测集 RBM-EVAL-OOD):

| 模型 | VOC Pearson r | Kendall τ(排序) |
|---|---|---|
| GVL(GPT-5-mini 当裁判) | 0.21 | 0.19 |
| VLAC-8B | 0.17 | 0.05 |
| TOPReward | 0.40 | 0.13 |
| RoboDopamine-8B | 0.80 | 0.45 |
| RoboReward-8B | 0.88 | 0.47 |
| **Robometer-4B** | **0.94** | **0.64** |

- 4B 参数打赢所有 8B 竞品;
- **通用 VLM 直接当裁判水平很差**:GPT-5-mini 方案相关性仅 0.21;RL-VLM-F 在"同任务不同质量"二选一上准确率 55.5%(接近抛硬币),Robometer 为 82.5%;"视频是否匹配指令"达 97.6%。侧面印证 WEAVER 甩掉 VLM 裁判的合理性——不只慢,判断力也不行。

**下游实战**:给 π0 做在线 RL(0.20 → 0.85);多阶段 RL(0.20 → 0.70);离线 IQL 比稀疏奖励高 2.4 倍;给世界模型 DreamZero 做规划打分(0.20 → 0.70,与 WEAVER 用法同款);零样本失败检测 F1 0.81(GPT-5-mini 仅 0.33)。

**弱点**(与 WEAVER 的抱怨对得上):"同质量偏好"准确率 82.5% 意味着约 1/6 判断出错;叠毛巾失败检测 F1 只有 0.58;只看视频(WEAVER 管线里仅右相机、1 fps),看不到力/触觉/精确深度,所以"笔插进杯子 vs 放在杯边"分不清。

**彩蛋**:WEAVER 致谢里专门感谢了 Jesse Zhang(Robometer 通讯作者)。另一个视角:Robometer 是 4B VLM、推理秒级,WEAVER 相当于**把这个 4B 模型对特定任务分布的判断力压缩进"池化 + 2 层 MLP"**,换来 0.0006 秒的打分速度。

## Q4: reward 在 WEAVER 中的作用是什么?

> 原问题:"reward 在 WEAVER 中的作用是什么"

**一句话:世界模型回答"接下来会发生什么",reward(+critic)回答"发生的这些事对任务是好是坏"——它是把 WEAVER 从"视频预测器"变成"可用于决策的模拟器"的判断模块。** 论文架构图称之为 **Latent Verifier(潜空间验证器)**。

**先划清两个边界**:

1. reward **不参与世界模型本身的训练**(动力学只用 flow matching 学预测,与奖励无关;reward head 是架在潜状态上的只读打分器);
2. reward **不通过 RL 梯度直接训练策略**(没有 Dreamer 式"想象中跑 actor-critic";用法全是打分 → 选择 → 过滤,策略更新只用模仿学习蒸馏被选中的轨迹)。

**三个下游中的三个角色**:

```
1. 策略评估 —— 自动进度读数
   真实动作序列 -> WM 想象 rollout -> reward 逐步打分 -> 进度曲线
   (让想象中的评估可自动化、可规模化,不需人工逐帧看或调 VLM)

2. 策略改进 —— 合成数据的质量闸门(最关键角色)
   策略采 B 条想象 rollout -> 算优势 A_hat -> 仅 A_hat > 0.1 的最优轨迹才蒸馏进策略
   (想象里既有真进步也有幻觉;没有这道闸门,"合成数据只差真实数据 4%"不可能成立。
    连"真实数据微调"对照组的数据也是同一套优势估计筛的——reward 同时是
    真实数据和合成数据的策展人)

3. 测试时规划 —— best-of-N 的裁判
   策略采 4 个候选动作块 -> WM 各想象一遍 -> 优势打分 -> 执行最高分
   (即 LLM 领域"生成器 + 验证器"的测试时扩展;效率角色在此最突出:
    潜空间打分 0.0006 秒,整个决策循环才能压进 1.45 秒)
```

**critic 是 reward 的延长线**:reward 只覆盖想象视野内(约 2.4 秒),critic 把打分延伸到视野外(任务长 40 秒):

```
A_hat = (视野内 reward 折扣和) + gamma^H * V(末端状态) - V(当前状态)
         reward head 提供        critic 兜底            基线,使分数可跨状态比较
```

**概念层面**:WEAVER 可拆成两种知识——世界模型 = **物理知识**(动作会让世界变成什么样,本身对成败没有意见);reward/critic = **任务知识**(什么状态算进展,经 CLIP 指令嵌入做到一个头服务所有任务)。没有 reward,WEAVER 只是高保真视频预测器;有了它,预测才接上选择与优化,评估、改进、规划三件事才全部成立。代价:系统判断力上限被蒸馏来源(Robometer)封顶,故用 0.1 阈值保守兜底,并把"更好的奖励模型"列为头号未来工作。
