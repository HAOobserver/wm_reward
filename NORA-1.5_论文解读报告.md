## 一、论文方法总览

### 1.1 一句话总结

NORA-1.5 = 预训练自回归 VLA (NORA) + 流匹配动作专家(架构升级)+ 用 V-JEPA2 世界模型和真值动作距离构造偏好数据、再用 DPO 做后训练(对齐升级),在仿真(SimplerEnv / LIBERO)和真机(Galaxea A1)上均达到 SOTA 级结果。

### 1.2 要解决的问题

现有 VLA 模型(π0、OpenVLA 等)依赖大规模模仿学习 + SFT,存在两个瓶颈:

1. **SFT 上限受限于示教质量**:模型难以超越示教数据,泛化性与可靠性不足;
2. **缺乏可扩展的奖励信号**:想借鉴 LLM 的 RL 后训练范式,但机器人领域真机 rollout 太贵、高保真仿真器难以搭建,reward 无处可来。

### 1.3 架构:NORA backbone + 流匹配动作专家

- **主干 (VL)**: NORA,3B 参数,基于 Qwen-2.5-VL-3B 在 Open X-Embodiment 上微调,FAST+ 分词器输出离散动作 token;
- **动作专家 ($\mathcal{A}$)**: 约 400M 参数的 flow-matching head,通过逐层 self-attention 读取 NORA 各 transformer 层的 Key/Value,直接回归 horizon $N=5$ 的连续动作序列:

$$
K_{VL,t},\ V_{VL,t} = VL_\theta(o_t, I), \qquad a_{t:t+N} = \mathcal{A}_\theta(K_{VL,t}, V_{VL,t})
$$

- **流匹配目标**: 对加噪动作 $a^\tau = (1-\tau)\,a + \tau\, a_0$(其中 $a_0 \sim \mathcal{N}(0,1)$,$\tau$ 为流匹配时间步)回归速度场 $v = a_0 - a$;
- **联合训练**: $L = L_{CE} + 10\, L_{FM}$(FAST token 交叉熵 + 流匹配损失),从 NORA-Long 继续训 150k 步,约 960 H100 小时;
- 细节:动作专家只允许 attend 图像和指令 embedding,屏蔽 FAST token 防止动作表征泄漏。

### 1.4 关键发现:流匹配不只是提速,还有相互增益

π0/π0.5 引入流匹配的动机是推理速度(自回归逐 token 解码太慢,流匹配几步去噪一次性生成整段动作)。本文发现联合训练带来**双向增益**:

- **正向 (VL → $\mathcal{A}$)**: 动作专家的条件输入是 NORA 逐层加工过的表征,已融合场景理解、指令语义和动作先验,比接在通用 VLM 上条件更好;
- **反向 ($\mathcal{A}$ → VL)**: 流匹配损失是**轨迹级监督**(一次回归整段 $N$ 步动作),梯度经 KV 连接反传回主干,迫使主干的内部表征编码连贯的多步计划,而非只关注下一 token。

**干净的对照证据**: NORA-1.5-FAST 与原版 NORA 使用完全相同的自回归 FAST 解码,唯一区别是主干经历过联合训练。SimplerEnv 视觉匹配零样本:NORA-Long 60.3% → NORA-1.5-FAST **74.0%**。解码器和模型规模都没变,涨幅只能来自主干本身变好。

**附带条件**: 该协同需要足够数据。动作专家随机初始化、无流匹配预训练,在真机小数据场景(50K 帧)下流匹配输出反而不如自回归解码,因此真机实验部署的是 NORA-1.5-FAST。

## 二、核心:世界模型作为打分器 (Reward Model)

### 2.1 为什么用世界模型打分

给策略采样的动作打分,有三条路:

| 路线 | 做法 | 问题 |
|---|---|---|
| 环境 rollout | 真机/仿真器执行后看结果 | 贵、慢、难扩展 |
| 动作距离 (GTA) | 与真值动作算距离 | 多条可行轨迹被压成一条;偏离示教流形后无指导 |
| **世界模型预测** | 预测动作后果,与目标比较 | **本文选择**;打分变成纯前向计算 |

用世界模型后,策略改进从"受真机时间限制"变成"受算力限制":几千条候选动作批量过一遍网络就全部打完分。

### 2.2 打分器结构与训练

打分器 = 冻结编码器 + 可训练预测器,共约 1.3B 参数(V-JEPA2-AC 结构):

- **编码器 $J$**: V-JEPA2 ViT-g,Meta 预训练权重,**全程冻结**。功能:图像 → 定长 latent embedding。性质:内容相近的图 embedding 相近,内容不同的图 embedding 距离远;
- **预测器 $P_\theta$**: transformer,**从头训练**,唯一被更新的部分。

$$
\hat{J}(o_{t+N}) = W_\theta(o_t, a_{t:t+N}) := P_\theta\big(J(o_t),\ a_{t:t+N}\big)
$$

输入当前观测的 embedding 和一段动作序列,一次前向直接输出 $N$ 步后观测的 embedding——不逐帧模拟,不生成像素,全程 latent space。

**训练方式(自监督,零标注)**: 从示教轨迹切转移三元组,输入"当前帧 embedding $J(o_t)$ + 示教动作块 $a^*_{t:t+N}$",监督信号是"真实未来帧过冻结编码器得到的 $J(o_{t+N})$",回归损失只更新 $P_\theta$。示教数据里"在 $o_t$ 执行 $a^*$ 后到达 $o_{t+N}$"是已发生的事实,未来帧是免费监督。

**关键局限**: 训练样本中的动作只有示教动作。WM 从没见过坏动作的后果(碰撞、抓空),对偏离示教流形的候选动作预测不可靠——而那些恰恰是最该被打低分的。这是公认的噪声源。

### 2.3 打分公式

**(1) WM-guided goal-based reward (Eq. 6)**

$$
R_g(a_{t:t+N}, o_t) = -\left\| J(o_g) - W_\theta(o_t, a_{t:t+N}) \right\|_1, \quad g \in \{\text{endgoal},\ \text{subgoal-}t\}
$$

其中 $J(o_g)$ 是**目标 embedding**(训练集真实帧过编码器),$W_\theta(o_t, a_{t:t+N})$ 是**预测终点 embedding**(WM 现算)。

- 两个量来源不同:**目标 embedding 来自示教数据集**;**预测终点不来自任何数据集**,是 WM 对每条候选动作分别预测的,K 条候选 → K 个预测终点;
- L1 距离(两向量逐位相减取绝对值求和)取负:预测终点离目标越近,reward 越高;
- **endgoal**: $o_g$ = 该 episode 末帧。度量预测状态到任务终态的 latent 距离,相当于 progress-based reward;
- **subgoal**: $o_g = o_{t+N}$,即示教轨迹 $N$ 步后的真实帧。预测 horizon 与监督 horizon 对齐,信号更干净。

**(2) Action-based reward / GTA (Eq. 7)**

$$
R_a(a_{t:t+N}) = -\left\| a^*_{t:t+N} - a_{t:t+N} \right\|_1
$$

候选动作与示教动作在 action space 的 L1 距离取负,不经过世界模型。

**(3) 组合 (Eq. 8)**

$$
R_{tot} = R_g + 0.5\, R_a
$$

互补逻辑:$R_g$ 提供目标导向的宽容性(容纳多条可行轨迹),$R_a$ 提供稳定锚点压制 WM 噪声。

### 2.4 完整打分链条

```text
示教数据取 (o_t, I)
    → policy 采样 K 条候选动作块(只采样,不执行)
    → 每条候选:
         WM 预测终点 embedding,与 goal frame embedding 算 L1 → R_g
         动作与示教动作算 L1                                → R_a
    → R_tot 排序 → 取 (winner, loser) 偏好对
    → DPO(此后分数丢弃,只留胜负标签)
```

## 三、DPO 后训练流程

### 3.1 数据来源:这不是 online RL

整个后训练阶段策略**从未与环境交互**。与三种范式对照:

| | 经典 online RL (PPO) | LLM 的 RLHF/DPO | 本文 |
|---|---|---|---|
| 状态从哪来 | 策略在环境里滚出来 | prompt 数据集 | **示教数据集** |
| 动作从哪来 | 策略执行 | 策略采样多条回复 | 策略采样多条动作块(**不执行**) |
| reward 从哪来 | 环境返回 | reward model | **WM + GTA 打分** |
| 环境交互 | 必需 | 无 | **无** |

结构与 LLM 的 RLHF 管线同构:示教状态 = prompt,候选动作块 = 候选回复,WM+GTA = reward model,DPO = 优化器。"执行后会怎样"这个环节被 WM 前向预测取代——这正是全文卖点。

### 3.2 六步管线

1. **SFT**: 在目标本体示教数据上微调 NORA-1.5。该模型身兼两职:DPO 初始化 + 冻结参考模型 $\theta_r$;
2. **训练打分器**: 用同一批示教数据训练 predictor $P_\theta$;
3. **采样候选**: 遍历示教数据 $(o_t, I)$,从 SFT 策略采样 K 条候选动作块;
4. **打分**: 每条候选算 $R_{tot}$;
5. **构造偏好对**: 排序取 $(a^W, a^L)$,形成偏好数据集 $\mathcal{D}$;
6. **DPO 训练**: 冻结参考模型,优化 DPO 目标,然后部署。

### 3.3 目标函数

**标准 DPO**(用于 FAST+ 自回归头,离散 token 似然可直接计算):

$$
\mathcal{L}_{DPO} = -\mathbb{E}\ \log \sigma\Big(\beta\big[\log\tfrac{\pi_\theta(a^W|x)}{\pi_{\theta_r}(a^W|x)} - \log\tfrac{\pi_\theta(a^L|x)}{\pi_{\theta_r}(a^L|x)}\big]\Big)
$$

直觉:相对参考模型,抬高 winner 的似然、压低 loser 的似然;margin 过 sigmoid 做 logistic 回归;$\beta$ 控制偏离参考模型的强度(隐式 KL 正则)。

**DPO-FM (Eq. 9)**(用于流匹配头): 流匹配头没有 tractable likelihood,标准 DPO 用不了。解法与 Diffusion-DPO 同思路——**用流匹配损失充当负对数似然的代理**(FM 损失越小 ≈ 似然越高)。记 $\ell(a) = \|\mathcal{A}(a^\tau, o_t, I, \tau) - v\|_2^2$ 为随机时间步 $\tau$ 上的流匹配 MSE,把四个 log-likelihood 全部替换为对应的 FM 损失:

$$
\mathcal{L}_{DPO\text{-}FM} = -\mathbb{E}_{\tau \sim U(0,1)}\ \log \sigma\Big(-\beta\big[\ell_\theta(a^W) - \ell_\theta(a^L) - \ell_{\theta_r}(a^W) + \ell_{\theta_r}(a^L)\big]\Big)
$$

四项依次为:策略拟合 winner、策略拟合 loser、参考模型拟合 winner、参考模型拟合 loser。

逐项含义:$\ell_\theta(a^W) - \ell_\theta(a^L)$ 要求策略在 winner 上的 FM 损失低于 loser;再减去参考模型的同一差值做**基准差分**,抵消样本本身难易度的影响(某条动作天生难拟合不会被误判为"被策略嫌弃")。

净效果:相对参考模型,策略把概率质量从 loser 挪向 winner,幅度被 KL 隐式正则锚住。DPO 只消耗**排序信息**,对 reward 绝对标定不敏感,因此可容忍 WM 打分噪声。

两个实现细节:流匹配专家用 DPO-FM,FAST+ 头用标准 DPO(结果带 `-FAST` 后缀);顺序是 预训练 → 本体 SFT → DPO,偏好优化是最后一道精修。

## 四、关键问答与讨论(阅读中的核心质疑)

### Q1: 打分公式里的 embedding("目标指纹")到底是什么?

图像编码器 $J$ 把一张图(约 15 万像素值)压成一个定长向量。该向量是图片**内容**的浓缩表示:内容相近 → 向量相近;内容不同 → 向量距离远。V-JEPA2 的预训练目标(从残缺视频帧预测完整帧特征)保证了向量抓住"有什么物体、在什么位置、什么状态",丢掉噪点纹理。

"目标 embedding" = 从示教数据集取出任务完成时刻的真实照片,过一遍编码器得到的向量。不比像素的原因:(1) 像素级比较对光照/抖动过敏,embedding 空间的比较发生在内容层面;(2) 预测器只能在 embedding 空间工作,预测向量比生成像素便宜得多。

### Q2: 如果任务某一步需要机械臂**远离**目标位置,得分会很低吗?

**endgoal 变体:会,这是它的固有缺陷。** endgoal 版 $R_g$ 是贪心的 progress 度量,假设正确动作总让状态单调逼近终态。绕行/避障/先退后进违反该假设。且偏好对在同一时刻的候选间做相对排序,所以这不是随机噪声而是**系统性打反标签**:绕行步,贪心错误动作被标 winner,正确绕行被标 loser。

论文实验抓到了该现象:SimplerEnv Move Near(要求避障)上,单独用 WM (endgoal) 做 DPO 后从 SFT 的 78.7% **掉到 73.9%**。作者解释:"purely goal-directed guidance does not account for the implicit safety constraints"。

**subgoal 变体:不会,这正是它存在的理由。** 比较目标 $J(o_{t+N})$ 是示教者 $N$ 步后实际到达的状态——若该步确实需要远离目标,示教轨迹在 $t+N$ 本身就处于"远离目标"的状态,排序方向自然正确。比较目标沿示教轨迹滑动,天然编码全部中间状态。**GTA 同理**:绕行步的真值动作本身就是远离动作。这解释了 WM (endgoal)+GTA 能把 Move Near 修回 88.0%。

附加两点:latent 距离 ≠ 末端执行器到目标的欧氏距离($J$ 编码整个场景状态);predictor 对 off-manifold 动作(如真的碰撞)预测本身失真。

### Q3: subgoal 是哪来的?

论文原文一句话:"The ground-truth subgoal image at time t is chosen as the t+N-th available frame." 即**示教视频里往后数 N 帧的那一帧**,零额外标注,零规划器。

一条遥操作 episode 的原始数据就是对齐的帧序列 + 动作序列。构造偏好数据时在轨迹上取锚点 $t$,所需材料全部现成:

| 材料 | 来源 |
|---|---|
| 当前观测 $o_t$ | 轨迹第 $t$ 帧 |
| 真值动作 $a^*$(GTA 用) | 轨迹第 $t$ 到 $t+N$ 步的记录动作 |
| **subgoal 图像** | **轨迹第 $t+N$ 帧,往后数 5 帧就是** |
| endgoal 图像 | 轨迹最后一帧 |

注意 "subgoal" 命名易误导:它**不是**语义子目标分解(不是"先抓取再移动再放置"),纯粹是时间意义上的短程目标——一个动作块长度后的示教状态。锚点 $t$ 沿轨迹滑动,每个 $t$ 有自己的 subgoal。

### Q4: RL 不是没有数据集、全靠 policy 自己 rollout 吗?训练数据哪来的?

本文后训练**不是 online RL,是离线偏好优化**。数据两个来源,分工明确:

- **示教数据集**(SFT 用的那批,真机 1000 条遥操作 episode):提供状态 $o_t$、指令、真值动作、goal frame、以及 WM predictor 的训练数据;
- **策略自己的采样**:给定示教状态,采样 K 条候选动作块——**只采样,从不执行**。传统 RL 中必须靠环境 rollout 获得的 reward,被 WM 前向预测取代。

代价(直觉正确的地方):**状态分布是示教分布,不是策略自身的访问分布**。偏好监督只覆盖示教者到过的状态;策略部署时一旦进入自己闯出的状态,那里没有任何偏好信号修正过它,WM 在那里的预测也不可靠。Online RL 恰恰在这点上更强——探索到哪学到哪。这是用"无需环境交互的可扩展性"换掉的东西。

### Q5: WM 预测的"终点"是从 SFT 训练集得到的吗?

不是。$R_g$ 里两个量来源不同:

- **目标 $J(o_g)$**: 来自训练集(示教轨迹的真实帧过冻结编码器);
- **预测终点 $\hat{J}(o_{t+N})$**: 不来自任何数据集,是 WM 对"这条候选动作执行后会到哪"的现场预测。候选动作是 policy 刚采样的、现实中从未执行过,数据集里不可能有它的后果。K 条候选 → K 个预测终点,与同一个目标比,谁近谁分高。

### Q6: reward 全锚定 SFT 数据,DPO 是否只是变相模仿、违背 RL 探索初衷?

**批评有相当合理成分**:GTA 锚定示教动作,subgoal 锚定示教状态序列,endgoal 理论上只锚定任务终态但 WM 的可信域被训练数据束缚;加上 KL 正则和离线单轮(无 GRPO 式 on-policy 迭代),探索半径确实很小,发现不了示教中不存在的全新策略。

**但"让动作分布和 SFT 一样"低估了它**,本质区别是**负梯度**:

- SFT 只有正样本,只说"这样做对",**从不说"那样做错"**,对模型自己的坏输出(抖动、抓偏、奔向干扰物)没有任何压低机制;
- DPO 的偏好对是**策略自己的采样**:winner 不是示教动作 $a^*$,是"模型自己输出中得分最高的那条"。DPO 同时抬高自己的好模式、显式压低自己的坏模式——对策略自身分布的一次重塑,而非收敛到示教分布。

**实验证据支持"分布修剪"而非"更用力背 SFT 数据"**:

1. 增益集中在 **unseen 任务**(+16%,seen 任务只微涨)——若只是拉回示教分布,收益应集中在 seen 任务;
2. 修的是 SFT 修不了的毛病:轨迹更平滑(抓取 9.7 → 7.0 个动作块)、之字形减少、抓干扰物 −4%——这些坏行为存在于示教数据点**之间**,正样本似然覆盖不到;
3. endgoal 项正是论文对该批评的回应("captures diverse feasible trajectories");且 GTA 单独用时在多数 unseen 任务上**低于不做 DPO 的基线**——消融本身演示了纯模仿锚定的危害。

**与 GRPO 的真正差距**: GRPO 能鼓励探索的前提是可验证 reward 在任意输出上可靠(数学/代码)。探索半径受 reward 可信域限制。机器人没有环境交互就没有 ground-truth 结果,WM reward 只在数据流形附近可信,所以只能做**保守的一步策略改进**——这是被 reward 质量逼出来的选择。真要走向 GRPO 式探索,缺的不是优化算法,而是 off-manifold 依然可靠的世界模型;WM 质量是这条路线的天花板。

准确定性:**在没有可靠环境反馈的约束下,把 RL 退化成一轮离线的、以自身采样为对象的分布修剪。** 比 SFT 多出负梯度和对自身失败模式的覆盖;比真 RL 少了探索。

| Galaxea A1 真机 | NORA-1.5-FAST 比 π0 / NORA 高 13%~46%;DPO 后成功率 56.9% → **70.0%**(+13%),unseen 任务 +16%,抓干扰物 −4% |

### 5.2 消融关键发现 (Tab. 4 / 5 / 6)

1. **endgoal 单独用会翻车**: Move Near 低于 SFT 基线 4.8%——纯目标导向看不见隐式约束(避障),且长时程 latent 预测不稳;
2. **subgoal 更稳**: 整体比 endgoal 高 1.7%;真机上 WM (subgoal)+GTA 全场最佳——短时程预测噪声小,真实环境的未见物体/多变条件下局部化引导更可靠;
3. **GTA 单独用在真机上有害**: 多数 "Put U in S" 任务低于基线——现实中多条轨迹可完成同一任务,强行贴合单一示教引入噪声;
4. **LIBERO 收益小**: SFT 基线已 94.5%,天花板效应;
5. **流匹配 vs 自回归与数据量相关**: 大数据(SimplerEnv 4M 帧)流匹配胜,小数据(真机 50K 帧)自回归胜——动作专家无流匹配预训练,适配需要更多数据。

综合结论:世界模型打分器的价值兑现程度,取决于**预测时程与其能力的匹配**——短程预测 (subgoal) 可靠,长程预测 (endgoal) 需 GTA 兜底。

## 六、方法定位与局限

**定位**: 与以往工作对比,V-JEPA2-AC / DINO-WM 把世界模型**在线用于规划**(MPC 式,推理时反复查询);本文把它**离线用作 reward model**,一次性生成合成偏好数据集,推理时完全不需要世界模型。RoboMonkey 虽做合成奖励但只用于测试时验证、不更新策略。本文给出了一条不依赖仿真器、不烧真机时间的 VLA 后训练路径,且 DPO 天然绕开流匹配头无 likelihood 的问题。

**局限**:

1. 打分质量天花板 = 1.3B predictor 的质量,有限数据适配,是公认噪声源(GTA 启发式因此无法去掉);
2. 状态分布锚定示教分布,无 on-policy 探索,off-manifold 无监督也无可靠打分;
3. 仅验证了 pick-and-place 类短程任务;
4. 引言里"用该打分器统一给 Open X-Embodiment 全量轨迹排序、转成巨型偏好数据集"的愿景,论文并未实施。
