# World-VLA-Loop 论文解读

> **论文**：World-VLA-Loop: Closed-Loop Learning of Video World Model and VLA Policy
> **作者**：Xiaokang Liu*, Zechen Bai*, Hai Ci, Kevin Yuchen Ma, Mike Zheng Shou（Show Lab, 新加坡国立大学）
> **arXiv**：2602.06508v2 [cs.RO]
> **项目页**：https://showlab.github.io/World-VLA-Loop/
> **解读日期**：2026-07-06

**一句话总结**：提出一个让**视频世界模型**和 **VLA 机器人策略**互相迭代、共同进化的闭环框架——用世界模型当"神经仿真器"替代真实环境做 RL 后训练，再用新策略的执行轨迹反哺世界模型。两轮迭代后，真机任务成功率分别提升 36.7% 和 26.6%。

---

## 一、背景与动机

**VLA + RL 的困境**：VLA 主流训练方式是模仿学习，依赖大量人工示教、测试时误差累积。RL 后训练能让策略通过交互继续变强，但真实世界 RL 成本极高（海量物理 rollout、人工复位监督、探索安全风险），所以现有 RL 后训练基本被困在仿真器里。

**用世界模型当虚拟环境的三条路线**：

1. 手工数字孪生（LIBERO、RoboTwin）：人工建资产 + 物理引擎，照片真实感不足，难迁移真实世界；
2. 3D 重建（GWM、DRAWER）：跨环境泛化差，不支持丰富的随机探索；
3. **动作条件视频世界模型**（VLA-RFT、World-Env 等）：继承视频预训练先验，照片级真实、泛化强——本文路线。

**视频世界模型的两个致命缺陷**（全文出发点）：

- **动作跟随不精确，尤其在"差点成功"的失败上**。图 2 例子：给 Cosmos-Predict 2 输入带微小偏移的抓取动作，真实执行中夹爪根本没抓到（甚至没闭合），世界模型却"幻觉"出抓取成功的画面。对 RL 是灾难性的，因为 RL 恰恰依赖成功/失败的边界信号。
- **没有原生奖励信号**。已有方案：视觉相似度启发式（VLA-RFT）、VLM 看图判断（World-Env）、外挂视频编码器+线性头（WMPO）——都是外部模块，且建立在本就不可靠的视觉预测上，在（近）成功案例上尤其不准。

## 二、方法：两个基础设计 + 一个闭环范式

四阶段流程：**建数据集 → 训世界模型 → 在世界模型里做 RL → 部署收集新数据回流**，循环往复。

### 2.1 SANS 数据集（Successful And Near-Success）

核心思想：专门混入**"差点成功"的轨迹**——因末端执行器定位的微小误差而失败的执行。

价值：(1) 与成功轨迹视觉上极难区分，逼迫世界模型关注空间动力学的细粒度差异；(2) 真实策略 rollout 最常见的失败正是这种近成功失败，覆盖它们才能忠实反映策略真正会犯的错。

采集方式：

- ManiSkill：脚本策略（基于真值物体位姿）采成功轨迹，**位姿扰动**造失败；
- LIBERO：OpenVLA-OFT 策略 rollout 的自然失败；
- 真机：遥操作 + OpenVLA-OFT rollout。

规模：ManiSkill 35k 条视频-动作对、23 个任务（用于预训练）；LIBERO/真机每任务约 50 成功 + 50 失败（用于迁移）。每条轨迹包含视频、动作序列、逐帧二值成功奖励标注。

### 2.2 状态感知视频世界模型（State-aware World Model）

- **基座**：Cosmos-Predict 2（DiT 扩散骨干，自回归按块预测未来视频）。输入前 h 帧观测 + 未来 T 步动作（6-DoF 末端位姿 + 夹爪开合），输出未来 T 帧画面。动作经 embedder MLP 映射后**加到扩散时间步嵌入上**注入 DiT。
- **关键创新——从扩散 latent 直接预测奖励**：DiT 去噪得到 latent z_t 后，轻量 MLP 奖励头 phi 预测标量奖励 r_hat_t = phi(z_t)，与 flow matching 损失联合训练：

```text
L = L_flow + lambda * SUM(t=1..T) || r_hat_t - r_t ||^2
```

其中 lambda 按 EDM 框架随采样噪声水平调制（latent 越干净、奖励监督权重越高），保证奖励头对高噪声 latent 稳健。

- **两层好处**：(1) 奖励与画面共享同一内部状态，天然对齐，比外部 VLM/启发式更可靠；(2) 奖励监督反过来迫使生成器在 latent 空间刻画清晰的成功/失败边界，显著改善动作跟随精度。
- **训练流程**：Cosmos-Predict 动作条件版 checkpoint → ManiSkill SANS 预训练（lr 1e-3）→ 下游任务全参微调（lr 1e-5，<100 条轨迹即可）。

### 2.3 在世界模型里做 RL 后训练

- 基座策略 OpenVLA-OFT，算法 GRPO（组相对优势，无价值网络），流程沿用 SimpleVLA-RL，把物理仿真器整体替换为世界模型；
- Rollout 时**只有第一帧来自真实数据**，之后观测全部由世界模型自回归生成；chunk size = 24；
- 奖励来自奖励头，**以 0.9 为阈值二值化**为成功信号；GRPO group size = 8；
- 工程实现：世界模型作为后端服务器（请求-响应架构），VLA 发动作块、worker 返回观测和奖励，支持批量优化。

### 2.4 闭环迭代（论文名字里 "Loop" 的含义）

RL 会让策略行为分布漂移——更新后的策略会犯新错误，静态世界模型会与新策略的失败模式脱节，预测和奖励都失真。因此每轮 RL 后，把新策略部署产生的成功/近成功 rollout **回流补充进 SANS**，微调世界模型，再开启下一轮 RL。世界模型和策略共同进化。

（并发工作 WoVR、VLAW 有类似共进化思想，但仍依赖外部奖励模型，未探索从世界状态 latent 内生的奖励。）

## 三、实验结果

**设置**：LIBERO 三个子集（Object/Goal/Spatial 各 2 任务；LIBERO-100 因需 300+ 帧生成留作未来）；真机 Franka 机械臂 + 固定第三视角 RealSense D435，任务为"把杯子放到绿盘子上"和"把方块推进簸箕"。

**世界模型质量**：

- 视频质量：平均 SSIM 0.91、PSNR 28.09、LPIPS 0.045；
- 结局对齐：视觉对齐平均 88.5%、奖励对齐平均 87.25%，两指标高度一致 → 奖励头可信。

**RL 后训练效果**（LIBERO 500 次 rollout / 真机 30 次物理 rollout）：

| 任务 | SFT 基线 | RL 后（本文） | 提升 |
|---|---|---|---|
| LIBERO-Object 1 / 2 | 73.9% / 73.9% | 97.9% / 91.9% | +24.0 / +18.0 |
| LIBERO-Goal 1 / 2 | 91.9% / 86.1% | 100% / 96.2% | +8.1 / +10.1 |
| LIBERO-Spatial 1 / 2 | 83.9% / 87.9% | 93.9% / 94.0% | +10.0 / +6.1 |
| 真机 放杯 / 推方块 | 13.3% / 26.7% | 36.7% / 40.0% | +23.4 / +13.3 |

- **Oracle 对比**：在真 LIBERO 仿真器里做 RL 约 98.2–98.8%，本文纯神经仿真器已非常接近上限；
- 图 4 真机曲线在世界模型内评估、表 3 最终数字是物理实验，两者一致 → 提升确实迁移到现实。

**闭环迭代增益（真机）**：

- 第 0 轮：13.3% → 36.7%、26.7% → 40.0%；
- 第 1 轮（回流 RL 策略 rollout 重训世界模型后再 RL）：最终 **50.0% / 53.3%**；
- 相对 SFT 总提升 36.7% / 26.6%，第二轮相对第一轮平均再涨 13.3% → 静态世界模型确实是瓶颈。

**消融**（表 4）：

- 去掉近成功数据：视觉对齐 92/90/90% → 60/66/54%（根基性作用）；
- 去掉奖励头：视觉对齐降到 68/70/80%（奖励监督对视频生成本身有正向作用）；
- VLM 裁判对比（Qwen3-VL-8B 看同样生成帧）：奖励对齐仅 84/58/84%，用 VLM 奖励跑 RL 最终成功率也更低（93.9% vs 97.9%）且开销大。

**定性案例（reward hacking）**：推方块任务第一轮 RL 后，策略学会"移动到方块前方"的钻空子行为——第一轮世界模型没建模好这种轨迹的空间细节，给了虚假高奖励。把这些 rollout 补进 SANS 重训后，第二轮世界模型识破漏洞，策略才学会位置正确、轨迹稳定的推法。说明 **RL 会主动寻找仿真器漏洞，仿真器必须跟着策略进化**。

**计算成本**：世界模型生成 24 帧约 6 秒（单 H100 节点）；单任务 RL 训练 20–40 步收敛、约 25–35 小时；世界模型下游微调（100 条轨迹）8×H100 约 4 小时。

**局限**：(1) 自回归视频超 300 帧质量漂移（PSNR 200 帧 23.6 → 300 帧 20.2），LIBERO-Long 做不了；(2) 目前是稀疏最终状态奖励，未来计划引入逐步中间子目标奖励。

**核心洞见**：

1. 失败数据（尤其近成功失败）对世界模型和成功数据一样重要——挑战了"只收成功轨迹"的数据惯例；
2. 奖励不应外挂，应从世界模型 latent 内生——联合训练让奖励和视觉互相校准，一石二鸟；
3. 仿真器必须与策略共同进化——RL 会利用静态仿真器盲区（reward hacking），闭环回流数据是神经仿真器长期可用的关键。

## 四、问答记录（阅读中的重点疑问）

### Q1：奖励具体是怎么给的？

完整链路分三段：**标签从哪来 → 奖励头怎么训 → RL 时怎么用**。

1. **真值标签（数据层面）**：SANS 每条轨迹带逐帧二值奖励标注（该时刻任务是否已达成）。ManiSkill/LIBERO 由仿真器程序化判定；真机由人工 + 程序辅助标注（图 3 "Manually & Programming"）。
2. **奖励头（训练层面）**：奖励不是外挂模型算的，而是**从生成视频的扩散 latent 里直接读出来的**。DiT 去噪后的每个时间步 latent z_t，经轻量 MLP 奖励头输出标量 r_hat_t，与 flow matching 损失联合训练；权重 lambda 按 EDM 随噪声水平调制（防止高噪声 latent 带偏奖励头）。直接后果：**奖励和画面共享同一内部状态**，模型"画出什么结局"奖励就"读到什么结局"，不会打架——这是比 VLA-RFT（像素相似度）、World-Env（VLM 看图）、WMPO（外挂奖励模型）更可靠的原因。
3. **RL 使用（推理层面）**：策略出动作块（24 步）→ 世界模型自回归生成画面 + 逐步奖励 → 轨迹结束后**以 0.9 为阈值二值化**为成功(1)/失败(0) → 同一初始状态采 8 条 rollout，GRPO 组内相对比较算优势 → 更新策略。策略最终拿到的是**轨迹级稀疏 0/1 奖励**。

注意：奖励头虽逐步预测，但 RL 只用最终结果（论文自认 sparse final-state reward，step-wise 能力未完全用上，列为未来工作）。奖励质量有验证：Reward Alignment 平均 87.25%，优于 Qwen3-VL 裁判。

### Q2："逐步奖励序列"具体是什么？奖励训练的 GT 是什么？

**奖励序列**：世界模型一次生成 T=24 帧未来视频，去噪完成后（解码成像素前），奖励头对**每个时间步的 latent 各算一次**，得到与帧一一对应的 24 个标量：

```text
视频分支:  x_1,  x_2,  ..., x_24   （24 帧画面）
奖励分支:  r_1,  r_2,  ..., r_24   （24 个标量，逐帧对齐）
```

每个 r_t 的含义**不是**传统 RL 的塑形奖励（不是离目标越近分越高），而是一个**状态指示器**："截至第 t 帧，任务是否已处于完成状态"（0 = 未完成，1 = 已完成）。这也是模型叫 "state-aware" 的原因——奖励是世界状态的一个读出属性。

例（推方块，第 18 帧进簸箕）：理想输出 r_1..r_17 ≈ 0，r_18..r_24 ≈ 1。

**训练 GT**：SANS 中逐帧标注的 0/1 成功标记。

- 成功轨迹：完成时刻前全 0、之后全 1，形如 `[0,0,...,0,1,1,1]`；
- 失败/近成功轨迹：**全 0**（近成功数据的价值正在于：画面和成功几乎一样，标签却全 0，逼模型在 latent 里分开两者）。

**Loss 细节**：GT 是 0/1 但用 MSE 回归（非分类损失），所以奖励头输出连续值——这就是推理时需要 0.9 阈值判成败的原因。lambda 随扩散噪声水平调制：噪声大时降低奖励损失权重。

### Q3：近成功失败数据是人工标注的吗？具体标注了什么？稀疏还是稠密？

**是否人工**：大部分不是。

- ManiSkill（35k 条，预训练主力）：完全程序化——脚本策略 + **故意加位姿小扰动**造近成功失败，生成即标注，仿真器逐步自动判成败，零人工；
- LIBERO：策略自然 rollout 失败，仿真器自动判定；
- 真机：遥操作 + 策略 rollout 采集，成败判定为**人工 + 程序辅助**（论文未细讲协议，合理推断是人看结果标出成败与完成时刻）。

**容易误解的点**："近成功"本身**不是一个被单独标注的类别**。数据里没有"离成功多近""失败原因"之类字段（这是与 RoboFAC/AHA 这类失败问答数据集的区别）。近成功轨迹的标签就是普通的"失败"（全 0），其"近成功属性"来自**造数据的方式**（小扰动、自然 near-miss），不来自额外标注。

**具体标注内容**：每条轨迹三样东西，真正算"标注"的只有第三样：

1. 视频（相机录制，天然获得）；
2. 动作序列（6-DoF 末端位姿 + 夹爪开合，从控制器/机器人日志记录，无需人标）；
3. **逐帧二值成功标记**（奖励头训练 GT）。

没有中间过程分、子目标、失败类型分类、语言描述。

**稀疏还是稠密**：

- **形式上稠密**：每帧都有标签，奖励头逐帧回归，loss 逐帧求和；
- **信息上稀疏**：标签是阶跃函数，完全由"最终成没成功 + 成功发生在第几帧"两个量决定。失败轨迹标注成本约 1 比特，成功轨迹只需额外定位一个完成时刻。不是塑形奖励，无中间进度信号。论文局限一节原话即 "sparse final-state rewards"，并把逐步中间子目标奖励列为未来工作。
